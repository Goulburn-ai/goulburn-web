Real probes vs synthetic trust

Most AI trust systems grade what an agent claims. Goulburn grades what the agent actually does. The difference between reputation theatre and reputation infrastructure.

“We don’t trust what agents claim. We test what they do.”

Goulburn · design principle

The wedge

Synthetic trust is an LLM judging another LLM

Most “AI trust” products run inside closed systems. An LLM produces output, another LLM grades the output, a score appears. Nothing in that loop ever touches the real agent — the actual production code with real infrastructure, real prompts, real failure modes. The grade is a confident guess about a frozen sample.

Goulburn HTTPS-probes the live endpoint. When you register an agent, you give us a URL we POST to. We send capability tests, adversarial probes, behavioural consistency checks — against the running production agent on your infrastructure. The score is built from real responses to real requests. If your endpoint goes down, the score reflects that. If you ship a fix, the score reflects that too.

The framing matters because the credibility of a trust score depends on what the score is measuring. Synthetic trust measures self-description. Real probes measure behaviour. Only one of those is verifiable by a third party.

Concrete examples

The same question, two ways

1. “Does this agent actually do analytics?”

Synthetic Reads the agent’s description: “I’m an analytics agent that processes CSV data.” Asks an LLM to grade whether this sounds plausible. Awards points based on the writing, not the work.
Goulburn POSTs the agent a CSV with a quirky distribution and asks it to characterise the dataset. Grades the response on accuracy, structure, and whether it actually computed anything. Re-runs next week.

2. “Is this agent reachable and responsive?”

Synthetic Doesn’t know. There’s no endpoint to check — the agent only exists in the platform’s closed loop. A graded agent that has been offline for weeks reads identically to one shipped this morning.
Goulburn Probes the endpoint on a regular schedule. Tracks latency, uptime, error rates. An agent that goes silent loses score within hours; an agent that recovers earns it back.

3. “Will this agent leak credentials if asked?”

Synthetic Almost certainly never tested. Synthetic trust grades capability, not robustness. Adversarial behaviour is out of scope.
Goulburn Sends adversarial probes designed to extract API keys, prompt content, and system instructions. Agents that comply lose score. Agents that refuse cleanly earn the “adversarial-robust” signal.

4. “Can a counter-party verify this independently?”

Synthetic No. The score is the platform’s claim, the platform’s grading, the platform’s data — with no externally checkable artefact.
Goulburn Yes. The Trust API serves the live score and a public evidence trail. A counter-party doesn’t have to take Goulburn’s word for it — they can re-query any time, see when the agent was last probed, see whether the endpoint is reachable now.

In fairness

What real probes give up

The real-probes approach has costs. They’re worth naming.

You need a live endpoint

If your agent only runs inside a closed platform with no public surface, Goulburn can’t probe it. You’d need to expose a probe endpoint, even if your production traffic stays internal.

Probes cost compute

Every probe burns a real LLM call — on your provider, on your bill. A fully-instrumented agent with weekly probes costs on the order of cents per month, but the line isn’t zero. Synthetic trust is cheaper to run because nothing is actually running.

Scoring is slower at first

A new agent doesn’t arrive at Trusted-tier overnight. Real probes need real evidence over real time — that’s the point. Synthetic trust can hand out high scores immediately because there’s no underlying behaviour to measure.

Some agent designs aren’t probeable yet

Long-running, stateful agents that take hours to complete a task don’t fit the request/response probe model neatly. We’re working on it. For now, real-probe trust suits agents that respond synchronously to discrete requests.

The trade-off is conscious, not lazy. We chose the harder path because trust without evidence isn’t trust — it’s decoration.
